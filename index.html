Kelola beberapa buku besar

Buat buku besar terpisah untuk kelompok/unit berbeda — masing-masing tersimpan sebagai dokumen Firestore terpisah, otomatis realtime di semua perangkat yang login.

Nama buku besar

Sinkronisasi Firebase (gratis)

Setiap transaksi tersimpan sebagai dokumen terpisah di Firestore milik akun Google Anda, tersinkron realtime ke semua perangkat yang login — tanpa risiko file tertimpa seperti pada penyimpanan berbasis satu file.

Cara setup Firebase (sekali saja, gratis)

  1. Buka console.firebase.google.com → Add project → beri nama bebas (mis. "Buku Besar") → selesaikan pembuatan project.
  2. Di menu kiri, klik Build → Firestore Database → Create database. Pilih lokasi server (mis. asia-southeast2/Jakarta), lalu pilih mode Start in production mode.
  3. Masuk ke tab Rules pada Firestore, hapus isinya, ganti dengan aturan berikut supaya data setiap orang hanya bisa diakses oleh pemiliknya sendiri, lalu klik Publish:
  1. Masuk ke Build → Authentication → Get started. Di tab Sign-in method, aktifkan provider Google.
  2. Klik ikon ⚙ (Project settings) di pojok kiri atas → scroll ke bawah ke bagian Your apps → klik ikon Web (</>) untuk mendaftarkan aplikasi web baru → beri nama bebas → Register app.
  3. Firebase akan menampilkan kode const firebaseConfig = {...}. Salin seluruh kode itu, tempel ke kotak di atas, lalu klik Simpan konfigurasi.
  4. Klik Masuk dengan Google. Selesai — data otomatis tersinkron di semua perangkat yang login dengan akun Google yang sama.
Gratis selamanya selama pemakaian di bawah kuota harian (20.000 penulisan/hari, 1GB penyimpanan) — jauh di atas kebutuhan pencatatan pribadi maupun kelompok kecil.
