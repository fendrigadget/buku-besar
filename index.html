Sinkronisasi Firebase (gratis)

Tombol Masuk dengan Google ada di pojok kanan atas halaman. Setiap transaksi tersimpan sebagai dokumen terpisah di Firestore, tersinkron realtime ke semua perangkat yang login.
Penting untuk fitur link publik: supaya orang yang belum pernah buka aplikasi ini juga bisa melihat laporan lewat link (tanpa setup apapun), Anda perlu tempel firebaseConfig yang sama ini langsung ke dalam kode sumber index.html Anda, di baris const BAKED_FIREBASE_CONFIG = null; menjadi const BAKED_FIREBASE_CONFIG = { ...config Anda... }; — ini aman dilakukan, config ini bukan rahasia, keamanan tetap dijaga oleh Security Rules.

Cara setup Firebase (sekali saja, gratis)

  1. Buka console.firebase.google.com → Add project → beri nama bebas → selesaikan.
  2. Menu kiri → Build → Firestore Database → Create database. Pilih lokasi (mis. asia-southeast2), mode production.
  3. Tab Rules pada Firestore, ganti isinya dengan aturan berikut (mendukung mode privat maupun publik per buku besar), klik Publish:
  1. Menu kiri → Build → Authentication → Get started → tab Sign-in method → aktifkan provider Google (isi Project support email) → Save.
  2. Masih di Authentication → tab Settings → Authorized domains → tambahkan domain GitHub Pages Anda (mis. ).
  3. Ikon ⚙ Project settings → bagian Your apps → ikon Web (</>) → Register app → salin kode firebaseConfig yang muncul.
  4. Tempel kode itu ke kotak di atas → Simpan konfigurasi → klik Masuk dengan Google di pojok kanan atas.
  5. Untuk fitur link publik, tempel juga config yang sama ke variabel BAKED_FIREBASE_CONFIG di kode sumber (lihat catatan di atas).
Gratis selamanya di bawah kuota harian Firestore (20.000 penulisan/hari, 1GB penyimpanan) — jauh di atas kebutuhan pemakaian pribadi/kelompok kecil.
